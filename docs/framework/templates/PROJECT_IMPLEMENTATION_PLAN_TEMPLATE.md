# Implementation Plan

Staged plan for project development.

Each stage = one coherent goal.

---

## Stage 0 — Setup / Foundation

Goal: prepare technical baseline.

Examples:

- Project setup
- Tooling
- Basic structure

Exit criteria:

- Project runs
- Tooling configured

---

## Stage 1 — Core Structure

Goal: establish base architecture and UI foundation.

Examples:

- Layout
- Design system
- Core components

Exit criteria:

- Stable base ready for features

---

## Stage 2 — Core Features

Goal: implement main user-facing functionality.

Examples:

- Primary user flows
- Key UI

Exit criteria:

- Main flow works end-to-end

---

## Stage 3 — Backend / Integration

Goal: connect to backend or external services.

Examples:

- Data persistence
- API integration

Exit criteria:

- Data stored and retrieved correctly

---

## Stage 4 — Secondary Features

Goal: supporting functionality.

Examples:

- Admin tools
- Notifications

Exit criteria:

- Supporting flows operational

---

## Stage 5 — Finalization

Goal: prepare for real usage.

Examples:

- QA
- Edge cases
- Performance sanity

Exit criteria:

- MVP usable in real conditions

---

## Rules

- Do NOT skip stages
- Do NOT mix multiple stages in one task
- Each stage must have clear exit criteria
- Tasks should map to a single stage
